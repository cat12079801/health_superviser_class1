# CLAUDE.md

本リポジトリで作業する際に従う規約をまとめる。グローバル設定の `~/CLAUDE.md` と併せて適用する。

## プロジェクト概要

第一種衛生管理者試験の学習を行う一問一答 Web アプリである。第一弾として関係法令科目のみを対象とする。バックエンドを持たず、素の HTML / CSS / JS と JSON データだけで完結する静的サイトとして構成する。デプロイ先は GitHub Pages を想定する。詳細は [REQUIREMENTS.md](REQUIREMENTS.md) に記載する。

## 開発フロー

- 初期実装以降のコード変更は、必ず作業ブランチを作成し Pull Request の形式で行う。`main` へ直接 push してはならない。人間がレビューを行うためである。
- ブランチ名はグローバル設定の命名規則（`<type>/<short-description>`）に従う。base branch 名（main 等）をそのまま使わない。
- `.github/pull_request_template.md` が存在する場合は、そのフォーマットに沿って PR を作成する。
- コミットメッセージおよび PR 本文は、である調で端的に記載する。

## 文体

- README・問題の解説文・ドキュメント等の文章では、意図した場合を除き体言止め（文末を名詞で終える表現）を使わない。文末は動詞・述語で終える。文章の正確性を担保するためである。
  - 例: 「〜を実装。」ではなく「〜を実装した。」と書く。
  - 選択肢・表のセル・見出しなど、体言で書くことが自然な箇所は対象外とする。

## 問題データ

- 問題は `data/` 配下のカテゴリ別ファイル（1ファイル＝1カテゴリ。ファイル名は `category` の値に一致させる。例: `data/law_hazardous.json`）に分割して管理し、読み込むファイルを `data/index.json` のマニフェストに列挙する。スキーマとファイル構成は [README.md](README.md) に記載する。
- `category` / `id` / `tags` の命名規則と全5科目への拡張方針は [docs/categories.md](docs/categories.md) に定める。新しい科目・カテゴリを追加するときはこれに従う。
- 各問題の `id` は学習履歴（localStorage）と紐づくため、既存の `id` は変更しない。
- 問題・解説は法令等を一次情報として作成したオリジナルとし、市販問題集の文面を転載しない。

### 難易度

- 新規に作成する問題の難易度は、**最低でも過去問相当**とする。過去問よりやや難しめは許容するが、過去問より易しい問題は作らない。
- 具体的には、保存年数・測定や健康診断の頻度・選任要件・数値基準など、過去問で問われる水準の知識の正誤を問う形式を基本とする。
- 五肢択一の正答位置は特定の番号に偏らせず、カテゴリ全体で分散させる。

#### 難易度メタ情報（`difficulty`）

- すべての問題は、難易度を表す `difficulty` フィールドを**必須**で持つ。値は次の3つのいずれかとする。検証スクリプト（[scripts/validate-questions.mjs](scripts/validate-questions.mjs)）でも必須・列挙値を検査する。
  - `"hard"`（難）: 過去問より難しい。複数条件の組合せ判断、紛らわしい数値・保存年数・頻度の区別、複数法令にまたがる、出題頻度の低いマイナー論点、新しい制度（化学物質の自律的管理等）。
  - `"standard"`（並）: 過去問相当。典型的な数値・要件の正誤を問う標準的な問題。判断に迷う場合の既定値とする。
  - `"easy"`（易）: 過去問より易しい。基礎的・常識的で、用語の定義をそのまま問う・明らかな誤りを1つ選ぶだけ等。
- 新規作成する問題は前項の方針（最低でも過去問相当）に従うため、原則 `"standard"` か `"hard"` を付す。`"easy"` は既存問題の分類のために用意した区分であり、新規には用いない。
- 学習状況画面では、カテゴリ別正答率の下に、この `difficulty` 別の正答率を細分化して表示する（[js/store.js](js/store.js) の `summarize` が集計し、[js/app.js](js/app.js) が描画する）。値（英語キー）と表示ラベル（難/並/易）の対応も `store.js` に集約する。

### コラム（`columns`）の追加・記述方針

解説の下部に表示する `columns` は、用語のイメージを補う補足である。次の基準に従って追加・記述する。

- **想定読者**: 一般的な高校生の語彙で理解できることを基本とする。日常で触れず、具体的にどのような物・作業かを想像しにくい用語を補足の対象とする。逆に、高校生が読んで意味が分かる一般的な語にはコラムを付けない。
- **語彙**: 専門用語・業界用語（例: 潜函工法、減圧症）をそのまま使わず、平易な言葉に言い換える。やむを得ず専門語を出す場合は、その場で意味を併記する。
- **内容**: 「それが何か」と「身近な具体例・どんな場面か」を1〜3文で示す。法令の細かい数値や手続きは解説本文に任せ、コラムは用語のイメージづくりに徹する。
- **個数**: 1問あたり任意個数とし、補足が不要な問題には付けない。1つの用語につき1コラムを目安とする。
- **文体**: 解説本文と同じく、である調で述語締めとする（[文体](#文体) の規則に従う）。`title` は「〇〇とは」のように体言で書いてよい。
