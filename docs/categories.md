# カテゴリ・科目の命名規則と設計

本アプリの問題データ（`data/` 配下の科目別ファイル）で用いる `category` / `id` / `tags` の命名規則を定める。
第一種衛生管理者試験の全5科目を対象とする。後方互換を保ちつつ拡張できるよう設計する。

## 1. 試験の科目構成

第一種衛生管理者試験は次の5科目（範囲）で構成される。本アプリの `category` はこの科目構成に対応させる。表示ラベルは、試験結果通知に記載される科目（又は範囲）の名称に合わせる。

| 表示ラベル | 科目（範囲）の正式名 | `category` | id 接頭辞 | 本番の問題数 | 配点 | 実装状況 |
|------------|----------------------|------------|-----------|--------------|------|----------|
| 法令（有害） | 関係法令（有害業務に係るもの） | `law_hazardous` | `law-haz` | 10問 | 80点 | 実装済み |
| 法令（一般） | 関係法令（有害業務以外のもの） | `law_general` | `law-gen` | 7問 | 70点 | 実装済み |
| 衛生（有害） | 労働衛生（有害業務に係るもの） | `hygiene_hazardous` | `hyg-haz` | 10問 | 80点 | 実装済み |
| 衛生（一般） | 労働衛生（有害業務以外のもの） | `hygiene_general` | `hyg-gen` | 7問 | 70点 | 実装済み |
| 労働生理 | 労働生理 | `physiology` | `phys` | 10問 | 100点 | 実装済み |

合計44問・400点満点である。合格基準は各科目40%以上かつ全科目合計60%以上である。

この表の機械可読な正準は [data/categories.json](../data/categories.json) とする。表示ラベル・id 接頭辞・データファイル名・実装フラグはすべて同ファイルに集約し、アプリと検証スクリプトはここを参照する（[5. 実装への影響](#5-実装への影響拡張時の作業) を参照）。新しいカテゴリを実装するときは、この表のラベルを用いる。

## 2. `category` の命名規則

- 基本形は `<subject>_<scope>` とする。
  - `subject`: 科目の系統を表す。`law`（関係法令）、`hygiene`（労働衛生）を用いる。
  - `scope`: 有害業務区分を表す。`hazardous`（有害業務に係るもの）、`general`（有害業務以外のもの）を用いる。
- 労働生理は有害業務区分による分割が存在しないため、例外として `scope` を付けず `physiology` とする。
- 値はすべて小文字スネークケースで記述する。
- 一度定義した `category` 値は変更しない。変更すると過去の学習履歴の集計が分断されるためである。

## 3. `id` の命名規則

- 形式は `<接頭辞>-<連番>` とする。接頭辞は前掲の表に従う（例: `law-haz`、`hyg-gen`、`phys`）。
- 連番はカテゴリ内で重複しない3桁のゼロ埋め通し番号とする（例: `hyg-haz-001`）。
- `id` は全問題で一意とし、いったん付与した `id` は変更・再利用しない。学習履歴（`localStorage`）と紐づくためである。
- 問題を削除する場合も、その `id` を別の問題に再利用しない。

例:

```
law-haz-001   関係法令（有害業務に係るもの）の1問目
hyg-haz-001   労働衛生（有害業務に係るもの）の1問目
phys-001      労働生理の1問目
```

## 4. `tags` の規則

- `tags` は表示および将来の絞り込み・分野別集計のための補助メタデータとする。出題ロジックや科目別集計には使わない。
- 自由な文字列配列だが、表記ゆれを防ぐため科目ごとに推奨語彙を定める。同義語は代表表記に寄せる。

| 科目系統 | 推奨する `tags` の語彙（例） |
|----------|------------------------------|
| 関係法令 | 安衛法 / 安衛令 / 安衛則 / 有機則 / 特化則 / 石綿則 / 粉じん則 / 酸欠則 / 高圧則 / 電離則 / じん肺法 / 労基法 / 女性則 / 事務所則 など法令名 |
| 労働衛生 | 作業環境管理 / 作業管理 / 健康管理 / 有害物質 / 局所排気装置 / 粉じん / 騒音 / 温熱条件 / 換気・採光 / 健康診断 / メンタルヘルス / 救急処置 などのテーマ |
| 労働生理 | 循環器 / 呼吸器 / 神経系 / 筋肉 / 感覚器 / 代謝 / 体温調節 / 腎臓・泌尿器 / 内分泌 / 血液 / 消化器 / ストレス など人体の系統 |

## 5. 実装への影響（拡張時の作業）

カテゴリのメタ情報は [data/categories.json](../data/categories.json) に一元管理する。1カテゴリ1要素の配列で、各要素は次のフィールドを持つ。

| フィールド | 内容 |
|------------|------|
| `category` | カテゴリ値（2. の命名規則に従う） |
| `label` | ホーム・統計・出題画面に表示する短いラベル（1. の表に従う） |
| `prefix` | id 接頭辞（3. の命名規則に従う） |
| `file` | データファイル名。`<category>.json` と一致させる |
| `implemented` | データを実装済みで出題対象とするなら `true`、設計のみで未実装なら `false` |

ブラウザ（[js/quiz.js](../js/quiz.js)）と検証スクリプト（[scripts/validate-questions.mjs](../scripts/validate-questions.mjs)）は、いずれもこのファイルを読む。ホームのカテゴリ別出題ボタンは [js/app.js](../js/app.js) が `implemented` なカテゴリから動的生成し、統計画面・正答率集計・出題画面のラベルもすべてここから導出する。したがって**新しい科目を実装するときの作業は次の2点で済む**。

1. `data/categories.json` の当該カテゴリの `implemented` を `true` にする（5科目は定義済み。表にない新カテゴリを足す場合は1要素追加する）。
2. `data/<category>.json` を作成し、問題を追加する。

メニューのボタン・統計・正答率は `categories.json` から導出されるため、UI やロジックのコード変更は不要である。`summarize` も `category` 単位で動的に集計する。科目数が大幅に増える場合は「科目 → カテゴリ」の2階層メニューへ再構成することを別途検討する。

## 6. データ分割の方針

- 問題データはカテゴリごとのファイルに分割して管理する。**1ファイル＝1カテゴリ**とし、ファイル名は `category` の値に一致させる（例: `data/law_hazardous.json`、`data/law_general.json`）。読み込むファイルは `data/categories.json` の定義から導出する。
- 起動時に `loadQuestions` が `data/categories.json` を読み、`implemented` が `true` のカテゴリの `file` を取得して結合する。新しいカテゴリを追加するときは、当該ファイルを作成し `data/categories.json` の `implemented` を `true` にする。
- データファイルは、そのカテゴリを実際に実装する PR（データ＋`categories.json` の `implemented` 切替）で作成・登録する。未実装カテゴリの空ファイルは事前に作らない。UI 未対応のまま空ファイルだけ存在すると、問題を追加しても表示されず誤解を招くためである。
- `id` は全ファイルを通して一意とする。これにより、分割しても学習履歴（`localStorage`）との整合は保たれる。
- カテゴリ単位で分割することで、`category` と `id` 接頭辞とファイルが1対1に対応し、別カテゴリを並行して編集してもファイル単位の衝突が起きない。

| `category` | ファイル |
|------------|----------|
| `law_hazardous` | `data/law_hazardous.json` |
| `law_general` | `data/law_general.json` |
| `hygiene_hazardous` | `data/hygiene_hazardous.json` |
| `hygiene_general` | `data/hygiene_general.json` |
| `physiology` | `data/physiology.json` |

## 7. 模擬試験モードでの利用（将来）

模擬試験モード（本番同様の44問・科目別足切り判定）を実装する際は、`category` を科目別の出題数と足切り判定の単位として利用する。有害業務区分（`hazardous` / `general`）と科目系統（`law` / `hygiene`）の組合せで、必要な出題構成を組み立てられるよう命名を設計している。
