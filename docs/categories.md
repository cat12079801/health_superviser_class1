# カテゴリ・科目の命名規則と設計

本アプリの問題データ（`data/` 配下の科目別ファイル）で用いる `category` / `id` / `tags` の命名規則を定める。
現在は関係法令の2区分のみを実装しているが、将来的に第一種衛生管理者試験の全5科目へ拡張することを見据えて設計する。実装済みの2区分との後方互換を保つ。

## 1. 試験の科目構成

第一種衛生管理者試験は次の5科目で構成される。本アプリの `category` はこの科目構成に対応させる。

| 科目 | `category` | id 接頭辞 | 本番の問題数 | 配点 | 実装状況 |
|------|------------|-----------|--------------|------|----------|
| 関係法令（有害業務に係るもの） | `law_hazardous` | `law-haz` | 10問 | 80点 | 実装済み |
| 関係法令（有害業務以外のもの） | `law_general` | `law-gen` | 7問 | 70点 | 実装済み |
| 労働衛生（有害業務に係るもの） | `hygiene_hazardous` | `hyg-haz` | 10問 | 80点 | 未実装 |
| 労働衛生（有害業務以外のもの） | `hygiene_general` | `hyg-gen` | 7問 | 70点 | 未実装 |
| 労働生理 | `physiology` | `phys` | 10問 | 100点 | 未実装 |

合計44問・400点満点である。合格基準は各科目40%以上かつ全科目合計60%以上である。

## 2. `category` の命名規則

- 基本形は `<subject>_<scope>` とする。
  - `subject`: 科目の系統を表す。`law`（関係法令）、`hygiene`（労働衛生）を用いる。
  - `scope`: 有害業務区分を表す。`hazardous`（有害業務に係るもの）、`general`（有害業務以外のもの）を用いる。
- 労働生理は有害業務区分による分割が存在しないため、例外として `scope` を付けず `physiology` とする。
- 値はすべて小文字スネークケースで記述する。
- 一度定義した `category` 値は変更しない。変更すると過去の学習履歴の集計が分断されるためである。

## 3. `id` の命名規則

- 形式は `<接頭辞>-<連番>` とする。接頭辞は前掲の表に従う（例: `law-haz`、`hyg-gen`、`phys`）。
- 連番はカテゴリ内で重複しない3桁のゼロ埋め通し番号とする（例: `hyg-haz-001`）。
- `id` は全問題で一意とし、いったん付与した `id` は変更・再利用しない。学習履歴（`localStorage`）と紐づくためである。
- 問題を削除する場合も、その `id` を別の問題に再利用しない。

例:

```
law-haz-001   関係法令（有害業務に係るもの）の1問目
hyg-haz-001   労働衛生（有害業務に係るもの）の1問目
phys-001      労働生理の1問目
```

## 4. `tags` の規則

- `tags` は表示および将来の絞り込み・分野別集計のための補助メタデータとする。出題ロジックや科目別集計には使わない。
- 自由な文字列配列だが、表記ゆれを防ぐため科目ごとに推奨語彙を定める。同義語は代表表記に寄せる。

| 科目系統 | 推奨する `tags` の語彙（例） |
|----------|------------------------------|
| 関係法令 | 安衛法 / 安衛令 / 安衛則 / 有機則 / 特化則 / 石綿則 / 粉じん則 / 酸欠則 / 高圧則 / 電離則 / じん肺法 / 労基法 / 女性則 / 事務所則 など法令名 |
| 労働衛生 | 作業環境管理 / 作業管理 / 健康管理 / 有害物質 / 局所排気装置 / 粉じん / 騒音 / 温熱条件 / 換気・採光 / 健康診断 / メンタルヘルス / 救急処置 などのテーマ |
| 労働生理 | 循環器 / 呼吸器 / 神経系 / 筋肉 / 感覚器 / 代謝 / 体温調節 / 腎臓・泌尿器 / 内分泌 / 血液 / 消化器 / ストレス など人体の系統 |

## 5. 実装への影響（拡張時の作業）

現状は関係法令の2区分のみを扱う。新しい科目を追加するときは、データ追加に加えて次のコード変更が必要になる。

- [js/quiz.js](../js/quiz.js) の `CATEGORY_LABEL` に、新しい `category` と日本語ラベルの対応を追加する。
- [index.html](../index.html) のホーム画面に、新しいカテゴリの出題ボタンを追加する。科目数が増える場合は「科目 → カテゴリ」の2階層メニューへ再構成することを検討する。
- 統計画面（カテゴリ別正答率）は `CATEGORY_LABEL` を反復して描画するため、`CATEGORY_LABEL` へ追加すれば自動的に集計・表示の対象となる。`summarize` も `category` 単位で動的に集計するため、追加のコード変更は不要である。

## 6. データ分割の方針

- 問題データは科目系統ごとのファイル（`data/law.json` / 将来の `data/hygiene.json` / `data/physiology.json`）に分割して管理する。読み込むファイルは `data/index.json` のマニフェスト（`files` 配列）に列挙する。
- 起動時に `loadQuestions` がマニフェストを読み、列挙された各ファイルを取得して結合する。新しい科目を追加するときは、当該ファイルを作成し `data/index.json` に登録する。
- `id` は全ファイルを通して一意とする。これにより、分割しても学習履歴（`localStorage`）との整合は保たれる。
- 関係法令は有害業務区分の両方（`law_hazardous` / `law_general`）を `data/law.json` にまとめる。区分は `category` で表現する。

## 7. 模擬試験モードでの利用（将来）

模擬試験モード（本番同様の44問・科目別足切り判定）を実装する際は、`category` を科目別の出題数と足切り判定の単位として利用する。有害業務区分（`hazardous` / `general`）と科目系統（`law` / `hygiene`）の組合せで、必要な出題構成を組み立てられるよう命名を設計している。
